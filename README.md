# 一念问道 · 文字修仙 Roguelike

一款纯前端、零依赖的 2D 文字修仙 Roguelike 网页小游戏。打开即玩，支持手机/桌面浏览器。

## 玩法简介

你是一名初入修行的散修，要在凶险的秘境中斩妖除魔、夺取机缘，一步步突破境界，最终渡过九重天劫、**飞升成仙**。每一次历劫都是一场随机生成的冒险，陨落不可避免，但「仙缘」会累积传承，让下一世的你走得更远。

### 核心系统

| 系统 | 说明 |
|------|------|
| **八种灵根** | 金/木/水/火/土/雷 + 稀有的混沌、剑灵根，开局随机，决定流派与加成 |
| **十大境界** | 练气 → 筑基 → 金丹 → 元婴 → 化神 → 炼虚 → 合体 → 大乘 → 渡劫 → 飞升，每境九层 |
| **回合制战斗** | 普攻 / 凝气 / 功法，含护盾、灼烧、中毒、冻结、麻痹、吸血、处决、剑意等机制 |
| **功法 16 种** | 黄/玄/地/天四阶，攻击、增益、治疗、削弱多种流派 |
| **法宝 13 种** | 被动叠加的装备，构筑专属流派 |
| **丹药 8 种** | 局内消耗品，战斗内外皆可服用 |
| **节点地图** | 战斗 / 精英 / 首领 / 奇遇 / 坊市 / 洞府，5 重秘境 |
| **随机奇遇** | 心魔、灵泉、遗府、丹炉等带分支抉择的事件 |
| **Roguelike** | 程序生成、随机掉落、永久死亡 + 元进度（仙缘传承）|
| **存档** | localStorage 自动存档，可随时继续 |

## 运行方式

纯静态文件，**无需构建**。

```bash
# 任选其一启动本地服务器
npx serve .
# 或
python -m http.server 8000
```

浏览器打开对应地址即可。也可直接将整个目录部署到任意静态托管（GitHub Pages / Vercel / Netlify / 对象存储 CDN）。

## 小游戏平台上架适配

- **屏幕**：以 460×920 的竖屏比例设计，`#game-frame` 自适应容器，移动端满屏、桌面端居中带边框。
- **离线**：无任何网络请求与外部依赖，资源全部本地化（emoji 作为图标，无图片包袱）。
- **存档**：使用 `localStorage`。若上架到微信/抖音小游戏，把存档读写替换为各自的 `setStorageSync/getStorageSync` 即可（集中在 `game.js` 的 `saveMeta/saveRun/loadRun/loadMeta` 四处）。
- **包体**：纯文本资源，体积极小（< 100KB），远低于各平台首包限制。
- **接入步骤（以微信小游戏为例）**：
  1. 新建小游戏项目，把本目录文件并入。
  2. 在 `game.js` 内将 4 处 `localStorage` 调用改为 `wx.setStorageSync / wx.getStorageSync`。
  3. 用 DOM→Canvas 适配层（如官方 `weapp-adapter`）或将渲染层迁移到 Canvas；当前 UI 为 DOM 渲染，H5 平台可直接上架，原生小游戏需做一层适配。

## 平衡性可调点

所有数值集中在 `js/data.js`：境界成长曲线（`REALMS`）、敌人倍率（`ENEMIES/BOSSES`）、掉落权重（`pickByRarity`）、商店定价（`SHOP_TIERS`）、仙缘消耗（`META_UPGRADES`）。修改即生效，便于运营调优。
